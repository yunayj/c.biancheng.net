Python标识符命名规范（新版）

简单地理解，标识符就是一个名字，就好像每个人都有属于自己的名字，它的主要作用是作为变量、函数、类、模块以及其他对象等的名字。

Python 中的标识符的命名不是随意的，而是要遵守一定的命名规则：
  • 标识符由字母（A~Z 和 a~z）、下画线和数字组成，但第一个字符不能是数字。
  • 标识符不能和 Python 中的关键字相同。
  • Python 的标识符不能包含空格、@、% 以及 $ 等特殊字符。

关键字是 Python 已经使用的标识符，所以 Python 不允许开发人员自己定义和关键字相同的标识符。

标识符除了有命名规则外，还有一些命名规范。

1) 见名知意

要起一个有意义的名字，尽量做到使人看一眼就知道是什么意思(提高代码的可读性)，例如将名字定义为 name，定义学生对象为 student 等。

2) 下画线命名法

多个单词之间使用下画线隔开，例如 user_name、student_info 等。

3) 驼峰命名法

类名所有单词首字母大写，其他字母小写；变量名、函数名所有单词中第一个单词小写，其他单词的首字母大写，例如类名 Student、UserView，变量名 userName、studentInfo 等。
